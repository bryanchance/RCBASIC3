GetProjectionGeometry(cam_dist, mA, f_vertex_count, ByRef columns, ByRef uv, graph_offset_x, graph_offset_y, v_color, ByRef vertex_count, ByRef vertex2D, ByRef index_count, ByRef index, ByRef clip_dist, ByRef min_x, ByRef min_y, ByRef max_x, ByRef max_y)

Calculates a 2D project from 3D vertices. It can calculate a triangle or quad projection based on the number vertices specified by f_vertex_count.

Parameters:
  cam_dist - Camera distance from clipping plane
  mA - A matrix containing the vertices to project
  f_vertex_count - the number of vertices ( NOTE: it must be 3 or 4 )
  columns - An array specifying the columns in the matrix for each vertex
  uv - Texture coordinates for each vertex (Note: must be given in the same order as the columns for the vertices)
  graph_offset_x - Where X is 0 ( generally the center of the view )
  graph_offset_y - Where Y is 0 ( generally the bottom of the view )
  v_color - The color for the polygon ( Note: This only matters if not texture is used when rendering )
  vertex_count - The number of vertices in the shape after clipping has been applied
  vertex2D - The returned vertices
  index_count - The number of indices returned after clipping has been applied
  index - The returned indices
  clip_dist - The returned minimum distance from the camera to the shape
  min_x - The minimum x value returned from the projection
  min_y - The minimum y value returned from the projection
  max_x - The maximum x value returned from the projection
  max_y - The maximum y value returned from the projection

NOTE: The vertex structure returned in vertex2D will be in the format the DrawGeometry() function needs
